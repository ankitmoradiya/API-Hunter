# Contributing to API-Hunter

Thank you for your interest in contributing to **API-Hunter**! We welcome all contributions, from code enhancements and bug fixes to documentation improvements and feature suggestions.

By participating in this project, you agree to abide by the project's code of conduct (if one is created).

### üêõ Reporting Bugs

If you find a bug, please help us by submitting an issue to our [GitHub Issues](https://github.com/ankitmoradiya/API-Hunter/issues).

When submitting a bug report, please include:
*   A clear and descriptive title.
*   The steps to reproduce the issue.
*   The expected behavior and the actual behavior.
*   The version of API-Hunter you are using.
*   Your operating system and Go version.

### üöÄ Suggesting Enhancements

We are always looking for ways to improve API-Hunter. If you have a feature request or an idea for an enhancement, please open an issue with the label `enhancement`.

### üíª Code Contributions

We follow the standard GitHub workflow for code contributions:

1.  **Fork** the repository to your own GitHub account.
2.  **Clone** the forked repository to your local machine.
    ```bash
    git clone https://github.com/your-username/API-Hunter.git
    cd API-Hunter
    ```
3.  **Create a new branch** for your feature or fix.
    ```bash
    git checkout -b feature/my-new-feature
    # or
    git checkout -b bugfix/fix-issue-123
    ```
4.  **Make your changes** and ensure your code adheres to the existing style and conventions.
5.  **Test your changes** to ensure they do not introduce new bugs.
6.  **Commit your changes** with a clear, descriptive commit message.
    ```bash
    git commit -m "feat: Add support for new CommonCrawl index format"
    ```
7.  **Push** your branch to your fork.
    ```bash
    git push origin feature/my-new-feature
    ```
8.  **Open a Pull Request (PR)** against the `main` branch of the original repository.

## ‚úÖ Pull Request Guidelines

*   Ensure your PR addresses a single issue or feature.
*   Reference the issue number in your PR description (e.g., `Closes #123`).
*   Keep your PR focused and avoid unnecessary changes.
*   Ensure all tests pass and the code builds successfully.

Thank you for helping to make API-Hunter better!
